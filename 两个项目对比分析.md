# 两个项目对比分析（对照"趣味智造"比赛要求）

## 比赛核心要求提取

从规则中提取的**评分关键词**：
1. 机械结构 — "必须具有一定的机械结构"
2. 智能控制 — "智能控制特征"
3. 人机交互 — "能与人友好交互"
4. 多种加工方式 — "3D打印机、激光雕刻机等"
5. 美观可爱 — "美观可爱的实物作品"
6. 趣味性 — "趣味娱乐"
7. 实用性 — "便利生活、使人方便"
8. 创意性 — "新奇、有趣、富有创意"

---

## 逐项对比

### 1. 机械结构 ⭐ 权重最高（"必须具有"）

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 机械部件数量 | **5 个**（2 驱动电机 + 刷子电机 + 吸尘风扇 + 万向轮） | 2 个（2 舵机） |
| 机构种类 | **驱动、清扫、吸尘、收集** 4 种不同机构 | 仅旋转展开/收起 1 种 |
| 评分 | ★★★★★ | ★★☆☆☆ |

> **清洁小车完胜。** 电机驱动底盘 + 旋转毛刷 + 负压吸尘 + 可拆卸收集盒，机械结构非常丰富。后视镜只有舵机转一下，机械含量偏单薄。

---

### 2. 智能控制

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 传感器 | 3 个悬崖传感器 + 1 个超声波 | 2 个超声波 |
| 控制逻辑 | 自主巡航 + 防坠落 + 避障 + 清扫路径 | 测距 + 接近判断 + 三级预警 |
| 语音/交互 | DFPlayer 播放可爱语音 | LD3320 语音识别控制 |
| 评分 | ★★★★☆ | ★★★★☆ |

> **持平。** 各有亮点——清洁小车有自主巡航（多传感器融合），后视镜有语音识别输入。

---

### 3. 人机交互 — "能与人友好交互"

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 交互方式 | 按钮启动 + 可爱语音播报（20 句台词） | 语音指令 + 按钮 + LED 预警 |
| "友好"程度 | **很高**（"出发咯！""好险差点掉下去！"） | 中等（功能性交互为主） |
| 现场展示互动 | **强**：放碎屑 → 按按钮 → 看它跑 → 听它说话 | 弱：需要有人从后面走才能演示 |
| 评分 | ★★★★★ | ★★★☆☆ |

> **清洁小车胜。** 比赛要求"友好交互"，清洁小车的可爱语音 + 自主行为 + 现场互动性远强于后视镜。评委能直接"玩"——撒碎屑让它清扫，小车边干边"说话"，非常有互动感。

---

### 4. 多种加工方式 ⭐ 明确鼓励

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 3D 打印 | 外壳、刷轴、收集盒、导风管 | 车把夹、传感器臂 |
| 激光切割 | 底板、顶盖面板（亚克力） | 不太需要 |
| 手工 | 热熔胶固定、焊接、扎带整线 | 热熔胶、扎带 |
| 种类数 | **3 种**（打印 + 切割 + 手工） | 2 种（打印 + 手工） |
| 评分 | ★★★★★ | ★★★☆☆ |

> **清洁小车胜。** 清洁小车天然需要更多零件，3D 打印 + 激光切割 + 手工三种加工方式都用得上。后视镜结构简单，激光切割用不太上。

---

### 5. 美观可爱

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 外观空间 | **大**：可以做成小动物/机器人造型 | 小：长条形传感器臂，造型受限 |
| "可爱"潜力 | **很高**：加眼睛、表情、颜色 | 低：功能性工具外观 |
| 评分 | ★★★★★ | ★★☆☆☆ |

> **清洁小车完胜。** 比赛明确要求"美观可爱"，清洁小车可以做成小企鹅、小瓢虫、机器人等造型，加 LED 眼睛会"说话"——天然适合"可爱"路线。后视镜是安全工具，很难做"可爱"。

---

### 6. 趣味性

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 观赏性 | **高**：看小车自己跑、扫、说话 | 中：看灯闪和臂展开 |
| 趣味感 | **像养了一个小宠物** | 像装了一个安全设备 |
| 评分 | ★★★★★ | ★★☆☆☆ |

> **清洁小车胜。** 比赛名字就叫"趣味智造"。清洁小车自主行为 + 可爱语音 = 天然有趣。

---

### 7. 实用性

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 解决的问题 | 桌面橡皮屑/纸屑清理 | 骑车盲区安全 |
| 实用价值 | 中等（特定场景） | **高**（安全问题，刚需） |
| 评分 | ★★★☆☆ | ★★★★★ |

> **后视镜胜。** 骑车安全是真正的刚需，后视镜解决的问题更"重要"。但清洁小车的场景更贴近小学生日常。

---

### 8. 创意性

| | 桌面清洁小车 | 自行车后视镜 |
|--|---|---|
| 新颖程度 | 中（有类似产品） | **高**（汽车盲区技术降维到自行车） |
| 故事性 | "帮小朋友打扫桌子的小管家" | "50 块做出 1500 块 Garmin 的功能" |
| 评分 | ★★★☆☆ | ★★★★★ |

> **后视镜胜。** "把汽车盲区检测 + 电动折叠后视镜移植到自行车"这个概念更新颖。

## 结论

### 推荐：桌面清洁小车

原因：比赛规则的 4 个高权重要求（机械结构、人机交互、多种加工、美观可爱），清洁小车全部拿满分。后视镜虽然创意和实用性更强，但在"机械结构""美观可爱""趣味性"三个硬指标上明显吃亏。

### 如果想增强清洁小车的创意分

可以把后视镜的 **语音识别** 特性嫁接过来：
- 加一个 LD3320 模块到清洁小车
- 对它说"开始打扫"→ 启动清扫
- 对它说"停下来"→ 停止
- 这样在"智能控制"维度再加一分，同时保持所有其他优势
