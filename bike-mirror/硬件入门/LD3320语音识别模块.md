# LD3320 语音识别模块

## 是什么

一个**能听懂人说话的芯片模块**，不需要联网，在芯片内部完成语音识别（离线识别）。你对它说"打开"，它就输出一个信号告诉 ESP32"用户说了打开"。

和 Siri / 小爱同学不同，它不能随便聊天，只能识别你**提前设定好的关键词**（最多 50 个）。但这对我们够了——只需要识别"打开"和"收起"两个词。

## 长什么样

```
大约 3cm × 2.5cm

    ┌────────────────┐
    │   LD3320       │
    │                │
    │  [芯片]  [晶振] │
    │                │
    │  [MIC] ←───────│── 自带小麦克风（有的版本需要外接）
    └────────────────┘
     VCC GND TXD RXD    ← 4 根主要引脚（串口版）
```

## 工作原理

```
人说话 → 麦克风拾音 → LD3320 芯片内部识别
                              ↓
                     匹配到关键词 "打开"
                              ↓
                     通过串口发送指令 ID（如 0x01）给 ESP32
                              ↓
                     ESP32 收到 0x01 → 执行展开动作
```

整个识别过程在 LD3320 芯片内部完成，不需要 WiFi、不需要云服务器，0.5 秒内出结果。

## 怎么接线

```
LD3320 模块          ESP32
┌──────────┐
│ VCC      │ ←── 5V（降压模块输出）
│ GND      │ ←── GND
│ TXD      │ ──→ GPIO 16（ESP32 RX2，接收 LD3320 发来的识别结果）
│ RXD      │ ←── GPIO 17（ESP32 TX2，发送配置指令给 LD3320）
└──────────┘
```

只需要 4 根线：电源 2 根 + 串口通信 2 根。

## 怎么设置关键词

LD3320 串口版通常有两种配置方式：

### 方式一：通过上位机软件配置（推荐）

1. 用 USB 转串口模块把 LD3320 连电脑
2. 打开厂家提供的配置软件
3. 输入关键词列表：

```
编号 01：打开
编号 01：展开     ← 同一个编号可以设多个同义词
编号 02：关闭
编号 02：收起
```

4. 下载到模块，断电重启即可

### 方式二：通过 ESP32 串口发送 AT 指令

```cpp
// 发送关键词设置（伪代码，具体 AT 指令看模块说明书）
voiceSerial.println("AT+KEYWORD=01,打开");
voiceSerial.println("AT+KEYWORD=01,展开");
voiceSerial.println("AT+KEYWORD=02,关闭");
voiceSerial.println("AT+KEYWORD=02,收起");
voiceSerial.println("AT+START");  // 开始识别
```

> 不同厂家的 LD3320 模块协议略有不同，买回来先看店家给的说明书和例程。

## 在自行车后视镜里怎么用

```
骑车人说：        LD3320 输出         ESP32 执行
"打开"     →     0x01          →    两侧舵机转到 90°，开始检测
"收起"     →     0x02          →    两侧舵机转到 0°，停止检测
```

代码核心：

```cpp
HardwareSerial voiceSerial(2);

void setup() {
    voiceSerial.begin(9600, SERIAL_8N1, 16, 17);
}

void loop() {
    if (voiceSerial.available()) {
        uint8_t cmd = voiceSerial.read();
        if (cmd == 0x01) {
            // 识别到"打开" → 展开臂
            extendArms();
        } else if (cmd == 0x02) {
            // 识别到"收起" → 折叠臂
            foldArms();
        }
    }
}
```

## 识别率和环境

| 环境 | 识别率 | 说明 |
|------|--------|------|
| 安静室内 | **90%+** | 比赛展示时完全没问题 |
| 普通街道 | 70–80% | 有些噪音干扰，多说几次 |
| 嘈杂马路 | 50–60% | 大风/大声噪音会干扰 |

**竞赛展示时是安静室内，识别率非常高。** 实际骑车时如果语音不方便，还有备用按钮可以手动切换。

### 提高识别率的技巧

- 关键词选 **2–3 个字**（太短容易误识别，太长容易识别不全）
- "打开"和"收起"刚好 2 个字，非常适合
- 说话时嘴离麦克风 **30cm 以内**
- 如果带风声，可以给麦克风贴一层薄海绵（防风）

## 和其他语音方案的对比

| | LD3320（本项目） | 百度语音 / 讯飞 | MAX9814 + ESP32 软件识别 |
|--|---|---|---|
| 是否需要联网 | **不需要** | 需要 WiFi | 不需要 |
| 识别词汇量 | 50 个预设词 | 无限 | 几个 |
| 价格 | **15–20 元** | 模块 30+ 元 | 麦克风 5 元 |
| 难度 | **简单**（串口通信） | 复杂（WiFi+API） | 很复杂（需要 AI 模型） |
| 适合场景 | **固定指令控制** | 智能对话 | 实验研究 |

LD3320 最适合"固定几个指令"的场景——不需要聊天，不需要联网，便宜又简单。

## 注意事项

- **买串口（UART）版**，不要买 SPI 版（SPI 版需要更多接线和代码）
- 有的模块**自带麦克风**，有的需要外接——买的时候确认
- 第一次用建议先用 USB 转串口 + 电脑上位机测试识别效果，确认能用了再接 ESP32
- 如果买的模块协议和代码对不上，联系卖家要例程
