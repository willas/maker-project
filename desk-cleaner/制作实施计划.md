# 智能清洁小车 — 制作实施计划

> 材料已到齐，这份文档规划从"零件摆桌上"到"比赛展示"的全过程。
> 每个步骤标注 👨 大人做 / 👦 小朋友做 / 👨‍👦 一起做。

---

## 总体原则

1. **先电子后结构** — 先在面包板/杜邦线上把电路跑通，再装进外壳
2. **先简单后复杂** — 一个模块一个模块加，每加一个都测试通过再加下一个
3. **先安全后功能** — 先验证悬崖检测（防掉桌子），再做清扫功能
4. **小朋友的参与不是"帮忙"，而是"主导局部任务"** — 给他们完整的小任务，有明确的成果

## 第一阶段：点亮 LED + 蜂鸣器 + 按钮（第 2 天，约 1.5 小时）

> 目的：让小朋友第一次体验"写代码 → 硬件动起来"的魔法。

### 👨 大人先做
- 电脑安装 Arduino IDE
- 添加 ESP32 开发板支持（首选项填 URL → 开发板管理器安装）
- 用 USB 线连接 ESP32，确认电脑能识别

### 👨‍👦 一起做
- 面包板上插好 ESP32
- 用杜邦线连接 LED 和蜂鸣器（参照引脚表）

```
接线（简单版）：
- 绿色 LED → GPIO 2 → 串联 220Ω 电阻 → GND
- 红色 LED → GPIO 4 → 串联 220Ω 电阻 → GND
- 蜂鸣器 VCC → GPIO 23，GND → GND
- 按钮 → GPIO 5 + GND（用内部上拉）
```

### 👦 小朋友主导
- **改 LED 闪烁频率** — 在 Arduino IDE 里改 `delay(500)` 的数字，看灯闪得快了还是慢了
- **改蜂鸣器音调** — 改 `tone(BUZZER_PIN, 1000)` 的频率数字，听声音变高变低
- **按按钮控制灯** — 按一下绿灯亮，再按一下红灯亮

#### 测试小程序（烧进 ESP32 试玩）

```cpp
// test_led_button.ino — 第一阶段测试程序
#define LED_GREEN  2
#define LED_RED    4
#define BUZZER_PIN 23
#define BUTTON_PIN 5

bool greenOn = false;

void setup() {
  pinMode(LED_GREEN, OUTPUT);
  pinMode(LED_RED, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  
  // 开机彩虹灯秀
  for (int i = 0; i < 3; i++) {
    digitalWrite(LED_GREEN, HIGH);
    tone(BUZZER_PIN, 1000 + i * 500, 200);
    delay(300);
    digitalWrite(LED_GREEN, LOW);
    digitalWrite(LED_RED, HIGH);
    delay(300);
    digitalWrite(LED_RED, LOW);
  }
}

void loop() {
  if (digitalRead(BUTTON_PIN) == LOW) {
    delay(50);  // 消抖
    if (digitalRead(BUTTON_PIN) == LOW) {
      greenOn = !greenOn;
      digitalWrite(LED_GREEN, greenOn);
      digitalWrite(LED_RED, !greenOn);
      tone(BUZZER_PIN, greenOn ? 1500 : 800, 200);
      while (digitalRead(BUTTON_PIN) == LOW) delay(10); // 等放开
    }
  }
}
```

### ✅ 这个阶段的成果
- 小朋友亲手让灯亮了、让蜂鸣器响了
- 理解"改代码 → 上传 → 硬件变化"这个循环
- **成就感爆棚，兴趣拉满**

## 第二阶段：让轮子动起来（第 3 天，约 2 小时）

> 目的：小车能前进、后退、转弯。

### 👨 大人做
- 接线 TB6612 电机驱动板 ↔ ESP32 ↔ 两个 N20 电机
- 供电：先用 USB 供电给 ESP32，电机暂时用 4 节 AA 电池（5 号电池盒）或直接接 18650 电池盒

```
TB6612 接线：
- AIN1 → GPIO 25, AIN2 → GPIO 26, PWMA → GPIO 32
- BIN1 → GPIO 27, BIN2 → GPIO 14, PWMB → GPIO 33
- STBY → GPIO 13
- VM → 电池正极 (6-7.4V)
- VCC → 5V, GND → 共地
```

### 👦 小朋友主导
- **把轮子装到电机上**（压上去就行）
- **调速实验** — 改代码里的速度数字，观察快慢变化
- **指挥小车** — "往前走！""转右！""倒退！"口头说，大人帮改代码上传
  - 后面可以简化为按不同按键控制

#### 测试小程序

```cpp
// test_motors.ino — 第二阶段测试程序
#define MOTOR_L_IN1  25
#define MOTOR_L_IN2  26
#define MOTOR_L_PWM  32
#define MOTOR_R_IN1  27
#define MOTOR_R_IN2  14
#define MOTOR_R_PWM  33
#define MOTOR_STBY   13
#define BUTTON_PIN   5

int step = 0;

void setup() {
  Serial.begin(115200);
  pinMode(MOTOR_L_IN1, OUTPUT);
  pinMode(MOTOR_L_IN2, OUTPUT);
  pinMode(MOTOR_R_IN1, OUTPUT);
  pinMode(MOTOR_R_IN2, OUTPUT);
  pinMode(MOTOR_STBY, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  
  ledcAttach(MOTOR_L_PWM, 5000, 8);
  ledcAttach(MOTOR_R_PWM, 5000, 8);
  
  digitalWrite(MOTOR_STBY, LOW);
  Serial.println("按按钮依次测试：前进→后退→左转→右转→停");
}

void driveMotors(bool l1, bool l2, bool r1, bool r2, int speed) {
  digitalWrite(MOTOR_STBY, HIGH);
  digitalWrite(MOTOR_L_IN1, l1);
  digitalWrite(MOTOR_L_IN2, l2);
  digitalWrite(MOTOR_R_IN1, r1);
  digitalWrite(MOTOR_R_IN2, r2);
  ledcWrite(MOTOR_L_PWM, speed);
  ledcWrite(MOTOR_R_PWM, speed);
}

void stopAll() {
  ledcWrite(MOTOR_L_PWM, 0);
  ledcWrite(MOTOR_R_PWM, 0);
  digitalWrite(MOTOR_STBY, LOW);
}

void loop() {
  if (digitalRead(BUTTON_PIN) == LOW) {
    delay(50);
    if (digitalRead(BUTTON_PIN) == LOW) {
      stopAll();
      delay(300);
      
      switch (step % 5) {
        case 0: Serial.println("前进"); driveMotors(1,0,1,0, 150); break;
        case 1: Serial.println("后退"); driveMotors(0,1,0,1, 150); break;
        case 2: Serial.println("左转"); driveMotors(0,1,1,0, 150); break;
        case 3: Serial.println("右转"); driveMotors(1,0,0,1, 150); break;
        case 4: Serial.println("停");   stopAll(); break;
      }
      step++;
      while (digitalRead(BUTTON_PIN) == LOW) delay(10);
    }
  }
}
```

### ⚠️ 安全提示
- 先把小车**架空**测试（轮子悬空），确认方向对了再放桌上
- 放桌上时**手跟着**，随时准备抓住防止掉落

### ✅ 这个阶段的成果
- 小车能走了！前进、后退、左转、右转
- 小朋友理解了差速转向原理（"划船"的比喻）

## 第三阶段：加装悬崖传感器 — 防掉桌（第 4 天，约 1.5 小时）

> 目的：这是安全关键环节。装完这个，小车就能安全在桌上跑了。

### 👨 大人做

- 接线：
  - 左 DO → GPIO 34，中 DO → GPIO 35，右 DO → GPIO 36
  - VCC → 5V，GND → GND

### 👦 小朋友主导
- **传感器实验** — 用手挡住/移开传感器，看串口打印"有桌面/没桌面"
- **边缘测试** — 慢慢把小车推向桌边，观察指示灯变化

#### 测试小程序

```cpp
// test_cliff.ino — 第三阶段测试程序
#define CLIFF_L  34
#define CLIFF_M  35
#define CLIFF_R  36
#define LED_GREEN 2
#define LED_RED   4

void setup() {
  Serial.begin(115200);
  pinMode(CLIFF_L, INPUT);
  pinMode(CLIFF_M, INPUT);
  pinMode(CLIFF_R, INPUT);
  pinMode(LED_GREEN, OUTPUT);
  pinMode(LED_RED, OUTPUT);
  Serial.println("悬崖传感器测试 — 用手挡住传感器试试");
}

void loop() {
  bool left   = digitalRead(CLIFF_L) == HIGH;
  bool middle = digitalRead(CLIFF_M) == HIGH;
  bool right  = digitalRead(CLIFF_R) == HIGH;
  
  bool danger = left || middle || right;
  
  digitalWrite(LED_GREEN, !danger);
  digitalWrite(LED_RED, danger);
  
  if (danger) {
    Serial.printf("⚠️ 悬崖! 左=%d 中=%d 右=%d\n", left, middle, right);
  } else {
    Serial.println("✅ 安全");
  }
  delay(200);
}
```

### 👨 大人做
- HC-SR04 装在车头前方，接线 Trig → GPIO 18，Echo → GPIO 19

### 👦 小朋友主导
- **测距实验** — 手放在传感器前面，来回移动，看串口打印的距离数字变化
- **"超声波尺子"游戏** — 先猜桌上某个东西离小车多远，再看传感器量出来的数字

#### 测试小程序

```cpp
// test_ultrasonic.ino — 第四阶段测试程序
#define TRIG_PIN 18
#define ECHO_PIN 19

void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  Serial.println("超声波测距 — 把手放在前面试试");
}

float measure() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  long dur = pulseIn(ECHO_PIN, HIGH, 30000);
  if (dur == 0) return 999.0;
  return dur * 0.034 / 2.0;
}

void loop() {
  float d = measure();
  Serial.printf("距离: %.1f cm", d);
  
  // 用字符画一个简单的"雷达"
  int bars = constrain((int)(d / 2), 0, 30);
  Serial.print("  |");
  for (int i = 0; i < bars; i++) Serial.print("=");
  Serial.println(">");
  
  delay(300);
}
```

### ✅ 这个阶段的成果
- 小车既能防掉桌，也能绕开桌上的杯子书本

## 遇到问题怎么办

1. **电机不转** — 检查 TB6612 的 STBY 是否接了 HIGH，电池有没有电
2. **传感器不灵** — 串口打印原始值看看，可能需要调传感器上的电位器
3. **DFPlayer 没声音** — 确认 SD 卡是 FAT32 格式，文件在 `mp3/` 文件夹里，编号是 4 位数字
4. **语音识别不响应** — 确认 TIANWEN BLOCK 里烧录成功，波特率对（115200）
5. **小车往桌边冲不停** — 检查传感器方向（要朝下），HIGH/LOW 逻辑对不对

每个阶段都有独立的测试程序，哪个模块出问题就单独测那个，不要在完整代码里找问题。
