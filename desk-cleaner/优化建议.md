## 🔴 高优先级（必须修复）

### 1. 语音识别串口协议解析

**问题：** `checkVoiceCommand()` 只读 1 字节，但 LU-ASR01 的串口协议通常是多字节帧（帧头 + 长度 + 指令 + 校验）。当前代码极可能读不到正确指令，语音控制完全不工作。

**修复方案：** 查阅 LU-ASR01 的串口协议文档，实现完整的帧解析：

```cpp
// 示例：LU-ASR01 常见协议格式
// 帧头(0xAA 0x55) + 数据长度 + 指令ID + 校验
uint8_t checkVoiceCommand() {
    if (!hasASR) return 0;
    if (asrSerial.available() < 4) return 0;  // 等待完整帧

    // 检查帧头
    if (asrSerial.read() != 0xAA) {
        while (asrSerial.available()) asrSerial.read();  // 清空缓冲
        return 0;
    }
    if (asrSerial.read() != 0xAA) {
        while (asrSerial.available()) asrSerial.read();
        return 0;
    }

    uint8_t len = asrSerial.read();
    uint8_t cmd = asrSerial.read();
    // 读掉剩余字节（校验等）
    for (uint8_t i = 0; i < len - 1 && asrSerial.available(); i++) {
        asrSerial.read();
    }

    return cmd;  // 返回 VCMD_CLEAN / VCMD_STOP / VCMD_HELLO
}
```

> ⚠️ 上面的帧格式是示例，请以 LU-ASR01 实际说明书为准。关键点是**不要只读 1 字节**。

### 2. `delay()` 阻塞导致无法及时停止

**问题：** `avoidEdge()` 和 `avoidObstacle()` 中用 `delay()` 做后退和转弯，期间无法响应"停下"语音或按钮。车在倒退/转弯时你喊"停下"它听不到。

**修复方案：** 改用状态机 + `millis()` 非阻塞计时：

```cpp
// 状态定义
enum CarState { IDLE, FORWARD, BACKING, TURNING };
CarState carState = IDLE;
unsigned long stateStartTime = 0;
int turnDirection = 0;  // -1=左, 1=右

void loop() {
    // 先检查停止指令（任何状态下都能响应！）
    if (buttonPressed() || checkVoiceCommand() == VCMD_STOP) {
        stopCleaning();
        carState = IDLE;
        return;
    }

    if (!cleaning) {
        delay(50);
        return;
    }

    // 状态机
    unsigned long now = millis();
    switch (carState) {
        case FORWARD:
            if (cliffDetected()) {
                carState = BACKING;
                stateStartTime = now;
                stopMotors();
                playRandom(SND_EDGE_1, 3);
                turnDirection = whichCliffSide();  // 根据传感器选方向
            } else if (getDistanceCm() < OBSTACLE_CM) {
                carState = BACKING;
                stateStartTime = now;
                stopMotors();
                playRandom(SND_OBS_1, 3);
                turnDirection = 1;  // 障碍物默认右转
            } else {
                forward(MOTOR_SPEED);
                maybeChatWhileCleaning();
            }
            break;

        case BACKING:
            backward(MOTOR_SPEED);
            if (now - stateStartTime >= BACK_TIME_MS) {
                carState = TURNING;
                stateStartTime = now;
                stopMotors();
            }
            break;

        case TURNING:
            if (turnDirection < 0) turnLeft(MOTOR_SPEED);
            else turnRight(MOTOR_SPEED);
            if (now - stateStartTime >= TURN_TIME_MS + random(300)) {
                carState = FORWARD;
                stopMotors();
            }
            break;
    }
}
```

好处：
- **任何时候都能响应"停下"**（安全）
- 主循环不会被阻塞
- 后退/转弯逻辑更清晰

## 🟡 中优先级（建议修复）

### 3. 超声波测距阻塞主循环

**问题：** `pulseIn()` 最多阻塞 30ms，期间悬崖检测和语音都暂停。

**修复方案：**
- 桌面场景不需要测太远，超时降到 15000（≈2.5m）：
```cpp
long duration = pulseIn(ECHO_PIN, HIGH, 15000);  // 最多 15ms
```
- 更好的做法：每隔几帧才测一次距离，不要每次 loop 都测：
```cpp
if (now - lastDistCheck >= 100) {  // 每 100ms 测一次
    lastDistCheck = now;
    currentDist = getDistanceCm();
}
```

### 4. 供电方案优化

**问题：**
- 2×18650 串联 7.4V → LM2596 降到 5V → ESP32 VIN（再降到 3.3V），两级降压，效率约 60%，发热大
- N20 电机标称 6V，接 7.4V 会过热缩短寿命

**修复方案（二选一）：**

**方案 A：单节电池 + 升压**
```
1×18650 (3.7V) → 升压模块 → 5V 给 ESP32 + DFPlayer
              → 3.7V 直接送 N20 电机（6V 电机在 3.7V 下也能转，只是稍慢）
```
- 最简单，但电机扭矩可能不够

**方案 B：保持串联，用 PWM 限压**
```
2×18650 (7.4V) → TB6612 VM 电机用（用 PWM 限制有效电压到 ~6V）
              → LM2596 降到 5V → ESP32 VIN + DFPlayer
```
- 代码中把 `MOTOR_SPEED` 最大值限制到约 210/255（等效 6.1V）

### 5. 左右悬崖分别处理

**问题：** 三个传感器 OR 在一起，无法根据"哪边到边缘"选择转向方向。

**修复方案：**

```cpp
int whichCliffSide() {
    bool left  = digitalRead(CLIFF_L) == HIGH;
    bool right = digitalRead(CLIFF_R) == HIGH;
    if (left && !right) return 1;   // 左边悬空 → 右转
    if (right && !left) return -1;  // 右边悬空 → 左转
    return random(2) == 0 ? -1 : 1; // 中间悬空 → 随机
}
```

配合状态机使用（见问题 2），不会再出现"左边到边缘还往左转"的情况。

### 6. 手动停止时先停车再播放语音

**问题：** `manualStop()` 先播放语音再 `stopCleaning()`，喊"停下"后车还会继续动 1-2 秒。

**修复方案：**

```cpp
void manualStop() {
    stopMotors();          // 先停车！
    brushOff();
    fanOff();
    digitalWrite(LED_GREEN, LOW);
    digitalWrite(LED_RED, HIGH);
    cleaning = false;
    playSound(SND_STOP);   // 再播语音
    Serial.println("清扫结束（手动）。");
}
```

### 7. ~~风扇吸力不足~~ ✅ 已解决

已将 3010 轴流风扇升级为 **5015 离心涡轮风扇（5V）**，负压提升 3-5 倍。

### 8. TCRT5000 传感器桌面颜色适应性

红外反射传感器对桌面颜色敏感。建议在 `setup()` 里加自校准：

```cpp
int cliffBaseline[3];

void calibrateCliffSensors() {
    Serial.println("悬崖传感器校准中...");
    for (int i = 0; i < 20; i++) {
        cliffBaseline[0] += analogRead(CLIFF_L);
        cliffBaseline[1] += analogRead(CLIFF_M);
        cliffBaseline[2] += analogRead(CLIFF_R);
        delay(50);
    }
    cliffBaseline[0] /= 20;
    cliffBaseline[1] /= 20;
    cliffBaseline[2] /= 20;
    Serial.printf("基准值: L=%d M=%d R=%d\n",
        cliffBaseline[0], cliffBaseline[1], cliffBaseline[2]);
}

bool cliffDetected() {
    int threshold = 300;  // 低于基准值这个数就算悬空
    return (analogRead(CLIFF_L) < cliffBaseline[0] - threshold)
        || (analogRead(CLIFF_M) < cliffBaseline[1] - threshold)
        || (analogRead(CLIFF_R) < cliffBaseline[2] - threshold);
}
```

> 注意：用模拟读取（analogRead）需要传感器模块用 AO 输出而非 DO 输出。

### 9. 清扫路径优化（比赛展示效果）

纯随机行走桌面大时覆盖不均。简单改进：
- 每 3-5 秒强制做一个微小的随机偏转（±5-10°），形成类似螺旋的覆盖

```cpp
// 在 FORWARD 状态中加：
if (now - lastSmallTurn >= 3000 + random(2000)) {
    lastSmallTurn = now;
    if (random(2) == 0) {
        ledcWrite(CH_L, MOTOR_SPEED * 0.5);
        delay(200);
        ledcWrite(CH_L, MOTOR_SPEED);
    } else {
        ledcWrite(CH_R, MOTOR_SPEED * 0.5);
        delay(200);
        ledcWrite(CH_R, MOTOR_SPEED);
    }
}
```

void logMsg(const char* msg) {
