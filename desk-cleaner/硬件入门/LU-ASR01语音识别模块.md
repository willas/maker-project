# LU-ASR01 语音识别模块

## 是什么

一个**离线语音识别芯片模块**，能听懂你说的话，不需要联网。你对它说"打扫"，它就通过串口告诉 ESP32"用户说了打扫"。

和 LD3320 是同类产品，但 LU-ASR01 是**升级版**——识别率更高、配置更简单、抗噪能力更强。

## 长什么样

```
大约 3cm × 2cm

    ┌────────────────┐
    │   LU-ASR01     │
    │                │
    │  [芯片] [MIC]  │ ← 自带麦克风
    │                │
    │  [TYPE-C]      │ ← Type-C 口直接烧录配置
    └────────────────┘
     VCC GND TXD RXD
```

## 和 LD3320 的对比

| | LU-ASR01 | LD3320 |
|--|---|---|
| 识别率 | **>95%** | ~90% |
| 响应速度 | **<200ms** | 较慢 |
| 自定义词条 | **数千条** | ~50 条 |
| 抗噪能力 | **强**（街道环境也行） | 弱（安静才行） |
| 配置方式 | **图形化拖拽**（TIANWEN BLOCK） | 上位机软件 |
| 烧录方式 | **Type-C 一键烧录** | 串口烧录 |
| 供电 | 3.3V–5V 都行 | 5V |
| 价格 | 25–48 元 | 15–20 元 |

## 在清洁小车里的角色

清洁小车有**两个语音模块**，分工不同：

```
LU-ASR01（耳朵）              DFPlayer Mini（嘴巴）
  "听"你说了什么                "说"可爱的回复
  Serial1: GPIO 21/22          Serial2: GPIO 16/17
         │                            │
         └──────── ESP32 ─────────────┘
```

| 你说 | LU-ASR01 识别 | ESP32 执行 | DFPlayer 播放 |
|------|-------------|-----------|--------------|
| "打扫" | → 发送 0x01 | → 开始清扫 | → "出发咯！" |
| "停下" | → 发送 0x02 | → 停止清扫 | → "我休息一下~" |
| "你好" | → 发送 0x03 | → 播放彩蛋 | → "你好呀！我是桌面小管家~" |

## 怎么接线

```
LU-ASR01        ESP32
VCC        ←──  5V（或 3.3V）
GND        ←──  GND
TXD        ──→  GPIO 21 (Serial1 RX)
RXD        ←──  GPIO 22 (Serial1 TX)
```

只需要 4 根线，和 LD3320 的接法一样。

## 怎么配置关键词

### 用 TIANWEN BLOCK 图形化工具（推荐）

1. 电脑下载安装 **TIANWEN BLOCK**（天问 Block，模块厂家提供）
2. Type-C 数据线连接 LU-ASR01 到电脑
3. 在软件里**拖拽积木块**设置关键词：

```
┌────────────────────────────────────┐
│ 当识别到 [打扫]  → 串口发送 [0x01] │
│ 当识别到 [开始]  → 串口发送 [0x01] │
│ 当识别到 [停下]  → 串口发送 [0x02] │
│ 当识别到 [停]    → 串口发送 [0x02] │
│ 当识别到 [你好]  → 串口发送 [0x03] │
└────────────────────────────────────┘
```

4. 点击**一键烧录** → 完成

不需要写代码，拖拽就行，小朋友自己就能配置。

### 测试

配置完后，对着模块说"打扫"，用串口监视器看有没有收到 `0x01`。

## 安装位置

麦克风需要朝外收音。装在小车顶部，外壳上开个小孔：

```
    ┌── 小车顶盖 ──┐
    │   (小孔) ←── 麦克风收音口
    │  [LU-ASR01]  │
    │  [ESP32]     │
    │  [DFPlayer]  │
    └──────────────┘
```

## 注意事项

- LU-ASR01 **自带喇叭驱动**，但本项目不用它播放语音（交给 DFPlayer），只用它的识别功能
- 第一次用先在电脑上单独测试识别效果，确认能识别了再接到 ESP32
- 串口波特率默认 **115200**（比 LD3320 的 9600 快）
- 如果识别率不够好，检查麦克风孔是否被挡住
